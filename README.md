# DataSaver (MVP)

DataSaver — это разрабатываемая распределенная система резервного копирования (СРК) корпоративного уровня (Enterprise) по образу и подобию архитектуры Micro Focus / OpenText Data Protector. 

На данном этапе реализовано **ядро системы (Data Engine)** и интерфейс командной строки (CLI) для локального резервного копирования и восстановления файловой системы Linux с компрессией «на лету».

## 🚀 Архитектурные особенности MVP

1. **Стриминг данных (Streaming I/O):** Программа не копирует файлы поодиночке. Вместо этого файлы упаковываются в непрерывный бинарный поток байт и записываются в единый архивный файл. Это предотвращает перегрузку дисковой подсистемы (I/O) при работе с миллионами мелких файлов.
2. **Компрессия «на лету» (ZSTD):** Данные сжимаются высокопроизводительным алгоритмом ZSTD (библиотека от Cloudflare) непосредственно в процессе чтения, что минимизирует нагрузку на CPU и экономит дисковое пространство.
3. **Модульная структура Go:** Логика бэкапа и восстановления строго изолирована. Приложение готово к бесшовному переходу на клиент-серверную архитектуру (gRPC) и расширению пула бэкаперов (базы данных, Windows VSS).

## 🛠️ Спецификация бинарного формата архива

Архивный файл `.dpbak.zst` представляет собой сжатый поток данных. Внутри ZSTD-оболочки развернута непрерывная бинарная лента, состоящая из блоков. Каждый файл упаковывается по следующему контракту:

| Смещение (байты) | Поле заголовка | Тип данных | Описание |
| :--- | :--- | :--- | :--- |
| `0..3` | `PathSize` | `int32` (BigEndian) | Длина строки пути файла |
| `4..11` | `Size` | `int64` (BigEndian) | Размер тела файла в байтах |
| `12..15` | `Mode` | `uint32` (BigEndian) | Права доступа Linux (POSIX Permissions) |
| `16..+PathSize` | `Path` | `[]byte` (строка) | Сам путь к файлу (например, `/etc/hosts`) |
| Сразу после пути | `FileData` | `[]byte` (поток) | Сырые байты содержимого файла (размером `Size`) |

После окончания `FileData` текущего файла сразу же начинаются первые 4 байта (`PathSize`) следующего файла. Поток завершается маркером `io.EOF`.

## 🔧 Требования и сборка

Для работы приложения необходим установленный компилятор **Go версии 1.18+** и ОС семейства **Linux / macOS** (для сохранения POSIX-прав файлов).

1. Скачайте зависимости:
   ```bash
   make prepare
   ```
2. Собрать media agent и disk agent:
   ```bash
   make build-all
   ```

## 💻 Инструкция по использованию (CLI)

Утилита поддерживает два взаимоисключающих режима: резервное копирование (`--backup`) и восстановление (`--restore`).

### 1. Резервное копирование (Backup)
Для создания полной резервной копии директории выполните:
```bash
./bin/diskagent --backup --src /path/to/source --archive my_backup.dpbak.zst
```
* **`--src`**: Целевая папка на сервере, которую нужно защитить.
* **`--archive`**: Имя и путь создаваемого архива (по умолчанию `backup.dpbak.zst`).

### 2. Восстановление данных (Restore)
Для восстановления файлов в альтернативную безопасную директорию (чтобы предотвратить случайную перезапись живой системы) выполните:
```bash
./bin/diskagent --restore --src my_backup.dpbak.zst --target /tmp/recovered_data
```
* **`--src`**: Путь к созданному ранее файлу архива.
* **`--target`**: Папка, куда будет развернута копия. Если флаг опущен, приложение попытается восстановить файлы по их **оригинальным** абсолютным путям.

### 3. Резервное копирование (Backup) на Media Agent
Для создания полной резервной копии Media Agent (сетевой):
```bash
./bin/mediaagent
```

```bash
./diskagent --backup --src /path/to/source --server ${ip}:50012
```
* **`--src`**: Целевая папка на сервере, которую нужно защитить.
* **`--server`**: IP сервера Media Agent.

## 🗺️ Вектор дальнейшего развития проекта (Roadmap)

- [+] **Сетевой слой (gRPC):** Разделение утилиты на `Disk Agent` (клиент на защищаемом сервере) и `Media Agent / Cell Manager` (сервер хранения).
- [ ] **Каталогизация (IDB):** Интеграция базы данных **PostgreSQL** для логирования сессий и мгновенного поиска файлов в бэкапах без чтения самих архивов.
- [ ] **Инкрементальный бэкап:** Определение изменившихся файлов по времени модификации (`mtime`) и размеру.
- [ ] **Cloud Storage:** Прямой стриминг бэкап-сетов в S3-совместимые облачные хранилища (MinIO, Yandex Cloud).
- [ ] **Application Agents:** Создание изолированных модулей для горячего резервного копирования баз данных (PostgreSQL, MySQL).
